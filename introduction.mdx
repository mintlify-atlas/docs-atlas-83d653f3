## What is Kamakiri?

Kamakiri is a powerful bootrom exploit tool designed for MediaTek-based devices, specifically targeting the Amazon Fire tablet "mantis" series. It enables device unlock, custom firmware installation, and low-level hardware access through a sophisticated multi-stage payload delivery system.

<Warning>
  This tool performs low-level hardware modifications and can permanently damage your device if used incorrectly. Only use Kamakiri if you fully understand the risks involved.
</Warning>

## How it works

Kamakiri uses a sophisticated multi-stage approach to exploit MediaTek bootrom vulnerabilities:

<Steps>
  <Step title="Bootrom Handshake">
    Establishes serial communication with the device's bootrom via USB
  </Step>
  <Step title="Stage 1 Payload">
    Loads initial exploit payload into bootrom memory
  </Step>
  <Step title="Stage 2 Payload">
    Deploys advanced payload with full eMMC and RPMB access
  </Step>
  <Step title="System Modification">
    Modifies partitions, downgrades firmware, and installs custom bootloader
  </Step>
</Steps>

## Supported devices

Currently, Kamakiri supports:

- Amazon Fire HD 8 (2018) - "karnak"
- Amazon Fire HD 8 (2020) - "mantis"

The tool specifically checks for the device type ID `AKPGW064GI9HE` to ensure compatibility.

## Architecture overview

Kamakiri consists of several key components:

- **Python modules**: High-level device communication and exploit orchestration
- **Shell scripts**: Wrapper scripts for common operations (bootrom, fastboot, recovery)
- **C payloads**: Low-level exploit code that runs on the device (stage1, stage2, lk-payload, microloader)
- **Binary assets**: Pre-built firmware components (preloader, lk, tz, recovery)
